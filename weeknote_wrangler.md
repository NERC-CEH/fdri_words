# Weeknote Wrangler

Every now and again, we produce a [weeknote](https://nerc-ceh.github.io/fdri_words/), about what has been happening in the fdri WP2 technical team. We rotate who does this in the team, the following is some hopefully helpful notes on the process.


## Steps
- create a new file in /weeknotes names {weeknumber}.md
- Write draft at the top [example starting point](https://github.com/NERC-CEH/fdri_words/commit/1721e8a60c909fa34b4bd05c3876c7046a7deb9e)
- This makes it public via https://nerc-ceh.github.io/fdri_words/weeknotes/2026/{weeknumber}.html but not findable
- Fill in updates about what has been happening and get everybody else's input
- At the end of the week tidy it up, remove the DRAFT title and add a link to it [here](https://github.com/NERC-CEH/fdri_words/blob/main/README.md)
- Once fully live on https://nerc-ceh.github.io/fdri_words/ , copy the page and send it out as an email to the mailing list


## Guidelines
Full creative freedom is left to the weeknote wrangler. These guidelines don’t need to be followed

- Each section should have a heading, emoji and a picture
- Don't keep it high level, dive into the detail
- Rough notes are fine, this is not a polished published output (weekly it too frequent to support that)
- Pictures don’t need to be pretty, even a screenshot of a terminal breaks up the text and helps with the flow of the document
