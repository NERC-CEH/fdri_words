## ğŸ›‚ New MQTT requirement

We've been asked by WP1/BGS if we can start supporting username/password MQTT instead of just client certs / mTLS which is what AWS IOT core supports by default. We are able to do this via a [custom authorizer ](https://docs.aws.amazon.com/iot/latest/developerguide/custom-authorizer.html), which we plan to roll out next year. This is to support lower powered devices which WP1 have struggled to find any that support certificate auth with MQTT.
