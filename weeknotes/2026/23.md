## Metadata Service

There is alot of work on the metadata side of things atm. Notable developments are:

- EA and SEPA station metadata has started to be collated and the metadata model setup to ingest these. This will allow us to deploy our new EA ingesters (we now have 4!) into the new accounts, alongside the existing SEPA one, and start collecting some data.
- Some fundamental changes to how our data processing configurations are handled.  As a reminder, any dataset (e.g. temperature, rainfall, potential evapotranspiration etc.) can have any number of "data processing configurations" that define "what to do" to generate that dataset. This can be QC methods, infilling, corrections, derivation calculations, or aggregations.  We now have explicit "data processing plans" saved to a dataset, so our pipeline has an explicit ordered list of operations it needs to carry out.  This has also required some work on the pipeline itself to handle these changes.  But the benefits are now we can be more flexible with ordering of configurations, and no longer need "interim" datasets defined for intermediate steps of processing.
